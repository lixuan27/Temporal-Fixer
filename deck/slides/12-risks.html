5 · Anticipated difficulties & contingencies

Every risk has a pre-registered fallback — and a trigger that fires it.

The fixer reverts the edit
target = source ⇒ "copy the source" shortcut		 observed CLIP-to-draft drops below 0.93 while flicker looks excellent; visible on watched video. Source-free conditioning (trained, not inference-only) + residual anchor to the draft. If it still reverts: styled target, so target ≠ source by construction.
Train–test flicker gap
reconstruction ≈2 vs real edit ≈7.4		 observed Character spectrum + amplitude measured per draft type on the same clips. Calibrated appearance-preserving jitter to the measured moments. Fallback: mix a small set of real mild-edit drafts with a warped-median pseudo-target.
Over-correction
training flicker stronger than deployment flicker		 observed Dynamic degree collapses (0.167) and imaging quality falls below the draft. Re-match the training degradation to the true draft statistics — this is the co-adaptation step, and it recovered motion, quality and edit preservation at once.
Per-frame quality ceiling
temporal smoothing softens detail		 bounded Imaging quality below the competitor at matched deflicker; five interventions already falsified. Accept and state the boundary: the ceiling is the editor's re-render, not the fixer. The lever is a sharper front-end editor (higher resolution, better distillation), not another loss weight.
Quantized editor kernel unavailable
the reference int4 build has no Hopper kernel		 blocked Build fails on our GPUs; verified across several build attempts. Substitute: the same 4-step distillation on full-precision weights — measured at 2.2 s/frame, sharpness 542 vs 636, flicker 7.80 vs 7.02. Acceptable. An FP8 Hopper-native build is being fetched in parallel.
Domain mismatch
reconstruction corpus is nature/outdoor, target is driving		 open Metrics hold on nature but degrade on driving clips. Rebuild the corpus on driving footage. Mitigating fact: the learned operation (appearance-preserving deflicker) is largely domain-agnostic, and rigid driving scenes flicker less than organic texture.
Long-horizon drift open Drift curve over minute-scale streams. Rolling cache + position re-indexing; a pinned frame sink if needed. The edit path carries no recurrent state, so any residual drift is base-rollout drift.
"It's just post-processing" — the reviewer's objection open — Beat naive flow smoothing on both axes at once (already shown), demonstrate cross-editor and cross-benchmark generalization with no retraining, and present the drift curve. That is a task, not a filter.
