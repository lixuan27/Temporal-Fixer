The generator

For every clean clip, run the same frozen editor per-frame under an appearance-preserving prompt ("keep the image exactly the same"). That gives a real, structure-exact, genuinely flickery draft. Train: draft → clean clip.

The flicker is produced by the deployment editor itself, so its character is right by construction — no synthetic corruption model to mis-specify.

The catch we measured

Identity reconstruction flickers at 2.07; the real reference edit flickers at 7.37. A full 16-cell sweep (prompt × guidance × seed) shows only a stronger prompt reaches ~7–9 — and a stronger prompt changes the appearance, which invalidates "target = source".

Guidance is a no-op here; seed policy moves flicker only ~1.25×. So prompt/config tuning alone cannot close the gap.

The resolution

Match the kind of flicker with real identity reconstruction; match the amount with calibrated appearance-preserving jitter (dialed to ≈7.4 and moment-matched to the measured character). Then train source-free, so no shortcut back to the source exists.
