Temporal Fixer

Image edits, video fixes. A per-frame image editor produces a flickery draft; a causal video model, trained on clean video only, fixes the temporal consistency.

Current experimental setup (reconstruction pipeline) — updated 2026-07-13
im2im editor: Qwen-Image-Edit + Lightning-v2 4-step distilled LoRA, 4 steps, true_cfg_scale=1.0, fresh seed per frame (the flicker source). The 4-step distillation is what keeps the reconstruction sharp & structure-preserving (an 8-step non-distilled base was under-denoised / distorted). Your exact nunchaku int4 2509 build has no H200/Hopper kernel — using the same Lightning distillation on full-precision weights; FP8-2509 next.
Training data: clean video + its per-frame im2im reconstruction (no paired-edit data, no synthetic degradation). target = clean source, draft = flickery reconstruction.
Fixer: Wan2.1 causal DiT, conditioned [draft | source], single-step, self-supervised.

2 · Reconstruction prompts tried

  • "Keep the image exactly the same, preserve all content, colors and structure"
  • "Reproduce this photo faithfully with the same appearance and details"
  • "Very slightly enhance sharpness and clarity, keep everything else identical"

Mild edits (color grade / overcast / vintage) used for the gap study; drastic edits (anime / Bauhaus / snow / watercolor) as the real-edit test.

3 · Train–test flicker gap

Flow-warped / raw flicker (mean |framet+1−framet|×100), 25 clips per set:

draft typeflicker
clean source~1.3real motion only
reconstruction (what we train on)7.6
mild edit (color grade, overcast…)7.2≈ reconstruction
drastic edit (anime, Bauhaus…)11.2higher

→ For mild / appearance-preserving edits (the Waymo case), reconstruction reproduces the editing flicker well — the flicker gap is small. Larger for drastic style edits.

4 · Why the editor must be structure-EXACT

Self-supervision sets target = clean source. That only teaches pure deflickering when the reconstruction stays close to the source (only temporal jitter differs). If the editor instead re-renders content each frame (draft ≠ source), the same objective teaches the fixer to reconstruct the source and would revert the edit at test. Hence the 4-step distilled, structure-preserving editor: it keeps every frame faithful so the fixer's only job is temporal consistency.
