Temporal Fixer

Image edits, video fixes. A per-frame image editor produces a flickery draft; a causal video model, trained on clean video only, fixes the temporal consistency.

Current experimental setup (reconstruction pipeline) — updated 2026-07-13
im2im editor: full-precision Qwen-Image-Edit, 8 denoising steps, true_cfg_scale=1.0, fresh seed per frame (the flicker source). The target 4-step distilled model (Qwen-Image-Edit-2509 int4 + Lightning v2, nunchaku) has no H200/Hopper CUDA kernel — FP8-2509 port in progress.
Training data: clean video + its per-frame im2im reconstruction (no paired-edit data, no synthetic degradation). target = clean source, draft = flickery reconstruction.
Fixer: Wan2.1 causal DiT, conditioned [draft | source], single-step, self-supervised.

2 · Reconstruction prompts tried

  • "Keep the image exactly the same, preserve all content, colors and structure"
  • "Reproduce this photo faithfully with the same appearance and details"
  • "Very slightly enhance sharpness and clarity, keep everything else identical"

Mild edits (color grade / overcast / vintage) used for the gap study; drastic edits (anime / Bauhaus / snow / watercolor) as the real-edit test.

4 · Decisive result — reconstruction fixer vs synthetic-degradation fixer

48 held-out OpenVE edit clips, VBench + flow-warped flicker + CLIP edit-preservation.

fixer trained onflicker ↓motion-smooth ↑subject-consist ↑edit preserved ↑imaging-Q
synthetic degradation6.130.9790.9180.9590.565
real reconstruction2.040.9950.9690.7420.469

The reconstruction fixer deflickers dramatically better (−67%, near a dedicated editor) — but it partially undoes the edit (CLIP 0.959→0.742). Cause: the 8-step editor re-renders the content (draft ≠ source), so with target = source the fixer learns to reconstruct the source and reverts the edit at test. This pins down exactly why the structure-EXACT 4-step distilled editor is needed: when draft ≈ source (only temporal jitter differs), target = source teaches pure deflickering instead of edit-reversion.

5 · Fixes in progress

  • Root cause: FP8-2509 (Hopper-native) → the true structure-EXACT 4-step distilled editor → low-drift reconstruction → retrain (expect strong deflicker and preserved edit).

  • Domain: switched training data from faces → outdoor/nature scenes; Waymo driving data next.
